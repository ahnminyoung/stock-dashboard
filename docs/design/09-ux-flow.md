# UX 흐름 및 개선 포인트

## 1. 사용자가 첫 3초 안에 보게 되는 것

### 1.1 초기 로딩 시퀀스 (0-3초)

```
[0초] 페이지 진입
  ↓
[0.1초] Header 렌더링 (로고, 검색창, 메뉴)
  ↓
[0.2초] Global Summary Bar 스켈레톤 표시
  ↓
[0.5초] 글로벌 요약 바 데이터 로딩 완료
        → 코스피, 코스닥, 나스닥, S&P500 실시간 지수 표시
        → 색상으로 상승/하락 즉시 인지
  ↓
[1초] 국내주식 섹션 스켈레톤 표시
  ↓
[1.5초] 국내 지수 카드 렌더링
        → 코스피/코스닥 상세 정보 + 미니 차트
  ↓
[2초] 국내 히트맵 렌더링 시작
      → 가장 눈에 띄는 "반도체 +5.2%" 박스 표시
      → 빨강/파랑 색상으로 시장 분위기 즉시 파악
  ↓
[3초] 급등주 TOP 5 표시
      → "삼성전자 +1.6%", "SK하이닉스 +2.3%" 등
```

**핵심 전달 메시지**:
- **0-0.5초**: "지금 코스피는 상승 중이고, 미국 시장도 상승 마감했구나"
- **0.5-1.5초**: "오늘 국내 시장은 반도체와 2차전지가 강세네"
- **1.5-3초**: "삼성전자가 오늘 많이 올랐구나, 관심 있던 종목 확인해야지"

---

### 1.2 시각적 우선순위

```
┌─────────────────────────────────────────────┐
│  [1] 글로벌 요약 바 (최상단 고정)             │ ← 시선 1순위
│  코스피 🔴 코스닥 🔵 나스닥 🔴              │    (3초 이내 반드시 확인)
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  [2] 국내주식 모아보기                       │ ← 시선 2순위
│  ┌──────────────────────────────────────┐   │    (3-10초)
│  │  국내 지수 카드 (4개)                 │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  ┌──────────────────┬──────────────────┐   │
│  │  [3] 히트맵      │  [4] 급등주      │   │ ← 시선 3순위
│  │  (테마맵)        │  (TOP 10)        │   │    (5-15초)
│  │  반도체 +5.2%   │  1. 삼성전자     │   │
│  │  2차전지 +3.8%  │  2. SK하이닉스   │   │
│  └──────────────────┴──────────────────┘   │
└─────────────────────────────────────────────┘
```

**색상의 역할**:
- 빨강(상승) vs 파랑(하락)으로 시장 분위기 **즉시** 파악
- 히트맵에서 진한 빨강 = "이 테마가 오늘 가장 핫하다"
- 숫자를 읽지 않아도 색상만으로 시장 상황 이해

## 2. 사용자 행동 시나리오

### 시나리오 1: 출근길 투자자 (모바일)

**시간**: 오전 8시 30분 (지하철 안)
**목적**: 전날 미국 시장 확인 + 오늘 국내 시장 예상

```
[1] 앱 진입 (5초)
    → 글로벌 요약 바 스와이프
    → "미국 나스닥 +0.7% 상승 마감"
    → "코스피 선물 +0.6% 상승"
    → 결론: 오늘 국내 시장 상승 예상

[2] 국내 히트맵 확인 (10초)
    → "반도체 +5.2%" 박스가 크고 진한 빨강
    → "AI/클라우드 +2.1%" 박스도 빨강
    → 결론: 기술주 강세

[3] 관심종목 확인 (5초)
    → 삼성전자 +1.6% (보유 중)
    → SK하이닉스 +2.3% (매수 고려)

[4] 앱 종료
    → 총 소요 시간: 20초
    → 핵심 정보 파악 완료
```

---

### 시나리오 2: 직장인 투자자 (데스크탑)

**시간**: 점심시간 12시 30분 (회사 PC)
**목적**: 장중 시장 점검 + 관심종목 확인

```
[1] 웹사이트 진입 (3초)
    → 글로벌 요약 바에서 "코스피 +0.58% 상승 중"
    → 시장 상태: 🔴 장중 (12:30)

[2] 국내 히트맵 인터랙션 (30초)
    → 히트맵에서 "반도체" 박스 호버
    → 툴팁: "삼성전자 +4.8%, SK하이닉스 +6.2%"
    → 박스 클릭 → 반도체 테마 상세 페이지 이동
    → 구성 종목 45개 전체 확인
    → 히트맵으로 돌아가기

[3] 급등주 TOP 10 확인 (20초)
    → "오늘 급등주는 반도체 관련주가 많네"
    → SK하이닉스 클릭 → 종목 상세 차트 확인

[4] 뉴스 확인 (15초)
    → "반도체주 강세 지속... SK하이닉스 신고가"
    → 뉴스 클릭 → 외부 링크 (새 탭)

[5] 관심종목에 추가 (5초)
    → SK하이닉스를 관심종목에 추가
    → 알림 설정 (126,000원 돌파 시)

[6] 웹사이트 북마크 (2초)
    → 총 소요 시간: 1분 15초
    → 점심시간에 시장 체크 완료
```

---

### 시나리오 3: 적극적 투자자 (데스크탑 + 듀얼 모니터)

**시간**: 오전 9시 (개장 직후)
**목적**: 실시간 시장 모니터링 + 매매 결정

```
[1] 웹사이트를 두 번째 모니터에 전체 화면으로 고정

[2] 실시간 데이터 모니터링 (지속)
    → 글로벌 요약 바: 1초마다 자동 갱신
    → 히트맵: 1분마다 자동 갱신
    → 급등주: 실시간 순위 변동 확인

[3] 테마 급변 감지 (알림)
    → "2차전지 테마 +5% 돌파" 알림
    → 히트맵에서 2차전지 박스 클릭
    → 관련 종목 즉시 확인

[4] 매매 결정 (1분)
    → 증권사 앱으로 전환하여 매수 주문
    → 다시 대시보드로 돌아와서 실시간 가격 모니터링

[5] 하루 종일 대시보드 유지
    → 시장 마감까지 백그라운드 모니터링
```

## 4. UI/UX 개선 포인트

### 4.1 즉시 적용 가능한 개선

| 항목 | 현재 문제 | 개선 방안 | 우선순위 |
|------|----------|----------|---------|
| 로딩 시간 | 첫 화면 로딩 3초 이상 | 스켈레톤 UI, 점진적 렌더링 | ⭐⭐⭐⭐⭐ |
| 히트맵 반응성 | 모바일에서 박스가 너무 작음 | 모바일은 카드형 리스트로 변경 | ⭐⭐⭐⭐⭐ |
| 색상 대비 | 색약자 구분 어려움 | 패턴 추가 (상승=줄무늬, 하락=점) | ⭐⭐⭐⭐ |
| 실시간 업데이트 | 새로고침 필요 | WebSocket 자동 갱신 | ⭐⭐⭐⭐ |
| 글로벌 요약 바 | 스크롤 시 사라짐 | Sticky 헤더로 항상 표시 | ⭐⭐⭐ |
| 뉴스 썸네일 | 텍스트만 표시 | 이미지 추가로 시각성 향상 | ⭐⭐⭐ |
| 히트맵 툴팁 | 호버만 지원 | 모바일은 탭으로 표시 | ⭐⭐⭐ |
| 관심종목 순서 | 고정된 순서 | 드래그 앤 드롭으로 재정렬 | ⭐⭐ |

---

- [ ] 호버 상태 개선
- [ ] 클릭 피드백 추가

**Phase 3: 접근성 개선 (1주)**
- [ ] 키보드 네비게이션 지원

- [ ] 색약자 모드

- [ ] 포틀릿 커스터마이징

### 4.3 모바일 UX 특별 고려사항

**터치 최적화**:
- 최소 터치 영역: 44x44px
- 스와이프 제스처 지원 (글로벌 요약 바, 히트맵)
- 롱 프레스로 상세 정보 표시

**성능 최적화**:
- 이미지 lazy loading
- 가상 스크롤 (긴 리스트)
- 오프라인 모드 지원 (캐시 활용)

**모바일 전용 기능**:
- 하단 탭바 고정 (홈, 국내, 해외, 관심, 더보기)
- 풀 다운 새로고침
- 햅틱 피드백 (알림, 가격 변동)

## 5. 앱/모바일 확장 고려사항

### 5.1 Progressive Web App (PWA)

**기능**:
- 홈 화면에 추가 가능
- 오프라인 모드 지원
- 푸시 알림
- 백그라운드 동기화

**구현 요소**:
```javascript
// manifest.json
{
  "name": "Stock Market Dashboard",
  "short_name": "주식대시보드",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#2563eb",
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

**장점**:
- 네이티브 성능
- 푸시 알림 (가격 알림)

## 6. A/B 테스트 아이디어

### 6.1 히트맵 vs 리스트

**A안**: 히트맵 (Treemap)
**B안**: 카드형 리스트

**측정 지표**:
- 클릭률 (CTR)
- 체류 시간
- 테마 상세 페이지 전환율

---

### 6.2 색상 시스템

**A안**: 빨강(상승) / 파랑(하락)
**B안**: 초록(상승) / 빨강(하락) (국제 표준)

**측정 지표**:
- 사용자 선호도 설문
- 오해/혼동 빈도
- 국가별 선호도 차이

---

### 6.3 글로벌 요약 바 위치

**A안**: 최상단 고정
**B안**: 헤더 아래 스크롤 가능

**측정 지표**:
- 스크롤 깊이
- 클릭률
- 사용자 피드백

---

## 9. 성공 지표 (KPI)

- **DAU**: 일일 활성 사용자 10,000명

- **페이지 뷰/세션**: 5페이지 이상

- **히트맵 클릭률**: 50% 이상
- **관심종목 등록률**: 70% 이상
- **뉴스 클릭률**: 30% 이상
- **검색 사용률**: 40% 이상

### 9.3 만족도

- **NPS 점수**: 40 이상
- **앱 스토어 평점**: 4.5/5.0 이상
- **재방문율**: 주 3회 이상
