### 3.1 데스크탑 레이아웃 그리드 시스템

```
전체 컨테이너: 최대 1920px
내부 컨텐츠: 최대 1680px (좌우 여백 120px)

그리드 시스템:
┌──────────────────────────────────────────────────────┐
│ [좌여백 120px] [컨텐츠 1680px] [우여백 120px]         │
└──────────────────────────────────────────────────────┘

컨텐츠 영역 내부 그리드 (12 Column):
│←─── 1/4 ───→│←─── 1/4 ───→│←─── 1/4 ───→│←─── 1/4 ───→│

포틀릿 크기:
- Full Width: 12/12 (전체)
- 2/3 Width: 8/12 (히트맵)
- 1/3 Width: 4/12 (탭 포틀릿)
- 1/2 Width: 6/12 (관심종목, 뉴스)
- 1/4 Width: 3/12 (지수 카드)
```

### 3.2 모바일 레이아웃 스택 순서

```
1. Header (고정, 60px)
2. Global Summary Bar (가로 스크롤, 120px)
3. 국내주식 모아보기 (세로 스택)
   - 섹션 헤더 (40px)
   - 국내 지수 2x2 그리드 (각 100px)
   - 국내 마켓맵 카드 (가변)
   - 급등/거래대금 TOP 5 (가변)
   - 관심종목 (가변)
   - 뉴스 (가변)
4. 해외주식 모아보기 (세로 스택)
   - 섹션 헤더 (40px)
   - 해외 지수 2x2 그리드 (각 100px)
   - 해외 섹터맵 카드 (가변)
   - 빅테크 모니터 (가변)
   - 관심종목 (가변)
   - 뉴스 (가변)
5. Bottom Tab Bar (고정, 64px)

전체 세로 길이: 약 3000~4000px (스크롤)
```

### 3.3 인터랙션 포인트

**데스크탑**:
- 호버 시: 카드 그림자 강화, 배경색 변경
- 클릭 시: 상세 페이지로 이동 또는 모달 표시
- 히트맵 박스: 호버 시 툴팁 (상세 정보)
- 탭: 클릭 시 즉시 전환 (애니메이션 0.2초)

**모바일**:
- 탭: 터치 시 즉시 전환
- 카드: 터치 시 하이라이트 → 상세 페이지
- 가로 스크롤: 스와이프 제스처 지원
- 하단 탭바: 항상 고정 (최소 터치 영역 48px)

## 4. 반응형 브레이크포인트

```css
/* Desktop Large */

  컨테이너: 1920px;
  컨텐츠: 1680px;
  히트맵: 2/3 (1120px);
  탭: 1/3 (540px);

@media (min-width: 1280px) and (max-width: 1919px) {
  컨테이너: 100%;
  컨텐츠: calc(100% - 120px);
  히트맵: 2/3;
  탭: 1/3;
}

/* Tablet */
@media (min-width: 768px) and (max-width: 1279px) {
  컨테이너: 100%;
  컨텐츠: calc(100% - 80px);
  히트맵: Full Width (세로 스택);
  탭: Full Width (세로 스택);

}

/* Mobile */
@media (max-width: 767px) {
  컨테이너: 100%;
  컨텐츠: calc(100% - 32px);
  모든 포틀릿: Full Width (세로 스택);
  지수 카드: 2x2 그리드;
  글로벌 요약 바: 가로 스크롤;
```

## 8. 와이어프레임 사용 시 주의사항

1. **정보 밀도**: 데스크탑에서는 정보 밀도를 높게, 모바일에서는 우선순위에 따라 선별 표시
2. **스크롤 깊이**: 데스크탑에서 글로벌 요약 바 + 국내 섹션 상단까지는 스크롤 없이 보여야 함
3. **터치 영역**: 모바일 터치 영역은 최소 44x44px 확보
4. **로딩 상태**: 각 포틀릿은 독립적으로 로딩되므로 스켈레톤 UI 필요
5. **에러 상태**: 데이터 로딩 실패 시 fallback UI 제공
6. **다크 모드**: 모든 컴포넌트는 다크 모드 대응 필요
